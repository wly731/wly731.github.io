Ruby Hashes vs. JavaScript Objects

Similarities and differences between Ruby and JavaScript

14 February 2016

As you progress to JavaScript from Ruby, you would probably realize that both languages actually share quite a bit of similarities. As long as you practice with the JavaScript syntax, you can actually apply the same logic from Ruby in JavaScript. In this post I will talk about one of the major differences between Ruby and JavaScript - Ruby Hashes vs. JavaScript Objects


Ruby hashes

In Ruby, you define a hash in this way: 

athlete = { height: 180, weight: 90, sport: basketball}

And you access different values like this:

athlete[:height] #=> 190
athlete[:weight] #=> 90
athlete[:sport] #=> "basketball"

In Ruby hashes declaration, we need to define both hash keys and the corresponding hash values. In the above example we use symbol as the data type of hash keys.


JavaScript Objects

In JavaScript, instead of using a hash, we create an object like this:

var athlete = { height: 180, weight: 90, sport: basketball};

And you access different values of the properties like this:

athlete.height / athlete["height"]   #=> 190
athlete.weight / athlete["weight"] #=> 90
athlete.sport / athlete["sport"] #=> "basketball"

As you may notice I have mentioned the word "property". Yes, in JavaScript everything is an object, and you can assign properties to different objects. In this case, instead of understanding it as a hash, we should rather say that we created an object called "athlete", which has 3 properties: "height", "weight" and "sport".
