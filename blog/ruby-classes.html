Ruby Classes

Class methods/variables vs. Instance methods/variables

7th February 2016

As we are getting more and more familiar with Ruby, another important concept that all newb Rubyists must understand is Ruby classes. A class is a blueprint that provides the behaviors (methods) to its objects. In other words, an object is an instance of a class. For example, "Ruby" is an object/instance of the string class.

Let's define a class and use it as an example to explore about different concepts of a Ruby class. 


    class Player
      @@sports = "basketball"

      def initialize(name, height, team)
        @name = name
        @height = height
        @team = team
      end

      def self.sports
        p @@sports
      end

      def profile
        puts "Player name: #{@name}"
        puts "Height(cm): #{@height}"
        puts "Current team: #{@team}"
      end

    end

Class variables vs. instance variables

For instance variables, the name is prefixed with an @. Each object has its own copy of the instance variables. While for class variables, the name is prefixed with @@. There should only be one copy for the same class. Using the example above, every player object created using this class has its own name, height and team. However, for all the players, the variable sports is the same, which is "basketball".


Class methods vs. instance methods

For instance methods, they are called on the object itself. For example, if we created a Player object called Baller_A and we call the instance method profile on it: 

Baller_A = Player.new("Stephen Curry",191,"GSW")

Baller_A.profile #=>

Player name: Stephen Curry
Height(cm): 191
Current team: GSW

For class methods, they are called on the class itself. To define a class methods we should follow this format: 
def self.method-name

  method content

end

For example, if we call the class method - sports on the Player class: 
Player.sports #=> "basketball"
